# PRD: 목표성과관리 시스템 (Performance Management System)

> 최초 작성일: 2026-03-21
> 최종 업데이트: 2026-03-22
> 참고 레퍼런스: 레몬베이스 (lemonbase.com)

---

## 1. 제품 개요

### 배경 및 목적

사내 목표 수립부터 연말 인사평가까지의 전 과정을 디지털화하여, 투명하고 공정한 성과관리 체계를 구축한다. 레몬베이스와 유사한 UX를 기반으로 자사 조직 특성에 맞게 커스터마이징한다.

### 핵심 문제

- 목표 수립·변경 이력 관리 어려움
- 평가 과정의 불투명성
- 조직 계층별 권한 관리 부재
- 조직평가 결과와 개인 인사평가 연동 수작업

---

## 2. 조직 계층 구조

```
최고관리자 (CEO/C-Level)
  └── 부문장 / 공장장 (조직담당 임원) ← 복수 존재 가능
        └── 팀장
              └── 팀원
```

**HR관리자**: 별도 역할 (조직 계층 외부, 시스템 운영 담당)

## 4. 핵심 기능 요구사항

### 4-1. 사용자 관리

- 직접 등록: Firebase Auth 즉시 생성 + 임시 비밀번호 발급
- 초대 링크: 7일 유효 토큰 생성 → 직원이 비밀번호 설정 후 활성화
- 엑셀 일괄 업로드: 신규 등록 + 기존 사용자 업데이트 (오류 행 표시)
- 활성화/비활성화: `isActive: false` 처리로 퇴사자 접근 차단

### 4-2. 목표 수립 (연초)

#### 목표 상태 흐름

```
DRAFT
  → PENDING_APPROVAL (팀원이 승인 요청)
    → LEAD_APPROVED (팀장 1차 승인)
      → APPROVED (임원 최종 승인 또는 팀장이 직접 확정)
        → IN_PROGRESS
          → COMPLETED
          → PENDING_ABANDON → ABANDONED
    → REJECTED (반려 → 팀원 재수정 가능)
```

#### 목표 데이터 모델


| 필드      | 설명                                          |
| ------- | ------------------------------------------- |
| 목표명     | 목표의 제목                                      |
| 세부추진내용  | 구체적인 실행 계획                                  |
| 추진기한    | 목표 완료 목표일                                   |
| 중요도/가중치 | 팀 내 목표 간 상대적 중요도 (%)                        |
| 상태      | DRAFT / PENDING_APPROVAL / LEAD_APPROVED / APPROVED / IN_PROGRESS / COMPLETED / PENDING_ABANDON / ABANDONED / REJECTED |
| 진행률     | 0~100%                                      |


### 4-3. 업무 진행 관리 (연중)

#### 진행상황 업데이트
- 목표별 진행률 (0~100%) 입력
- 업무 메모/코멘트 추가
- 상태 변경: 진행중 / 완료 / 포기
- 포기/완료도 팀장 승인 필요

#### 1on1 관리
- 팀장-팀원 간 1on1 채팅 Q&A 형식
- 질문/답변 스레드 기록
- 1on1 히스토리 조회

### 4-4. 조직평가 관리 (연말)

#### 조직평가 등급 지정
- CEO가 부문/공장(DIVISION/HEADQUARTERS) 조직 등급 직접 지정 (즉시 APPROVED)
- 등급 변경 이력 자동 기록 (`orgGradeHistories`)

#### 개인 평가등급 할당 제한 (쿼터제)
- HR관리자가 조직별 개인 등급 쿼터 확정
- 전역 비율 설정 기반 자동 계산 (총 인원 × 비율, 합계 보정)
- 산하 하위 조직 포함 총 인원 재귀 계산
- CEO 등급 변경 시 기확정 쿼터 자동 DRAFT 초기화

#### 쿼터 예시


| 조직평가    | A   | B   | C   | D   | E   |
| ------- | --- | --- | --- | --- | --- |
| A (최우수) | 1명  | 3명  | 5명  | 1명  | 0명  |
| B       | 0명  | 2명  | 6명  | 1명  | 1명  |
| C       | 0명  | 1명  | 7명  | 1명  | 1명  |


#### 개인 평가등급 부여 흐름

```
CEO: 부문/공장 조직 등급 지정 (즉시 확정)
  → HR관리자: 조직별 개인 쿼터 확정
    → 팀장: 팀원 평가등급 추천 → 임원에게 요청
      → 임원: 최종 등급 확정 (EXECUTIVE_CONFIRMED)
        → HR관리자: 일괄 공개 (PUBLISHED)
```

### 4-5. 마일리지 관리
- HR관리자/CEO가 전체 사용자 마일리지 조회
- 마일리지 수동 조정 + 메모 기록
- 대시보드: 개인 마일리지 + 등급 표시

### 4-6. 연간 목표 관리
- 회사 연간 목표 등록/수정
- 조직별 연간 목표 등록/수정
- 대시보드 배너에 회사/조직 연간 목표 표시

## 8. 성공 지표 (KPI)

- 목표 등록률: 전 직원 대비 목표 수립 완료 비율 90% 이상
- 승인 처리 시간: 목표 승인 요청 후 3 영업일 이내 처리
- 평가 완료율: 평가 기간 내 평가 완료 100%

---

## 9. 제약 조건 및 가정

- 1인당 목표 수는 최소 1개, 최대 10개로 제한
- 가중치 합계는 팀원 전체 목표 기준 100%
- 평가 사이클은 연 1회 (1월~12월) 기준, 추후 반기/분기 확장 고려
- 조직 계층은 최대 4단계 (회사 → 부문/공장 → 팀 → 팀원)

---

## 10. 미해결 질문 (Open Questions)

- 목표 수립 시 상위 조직 목표와의 연계(Alignment) 기능 필요 여부
- 다면평가(360도 평가) 도입 여부
- 평가등급 직원 공개 시점 및 방법 (일괄 공개 vs 개별 통보)
- 모바일 앱 별도 개발 여부 (vs 모바일 웹)
- 기존 HR 시스템(ERP 등)과의 데이터 연동 필요 여부
- 알림 방식: 이메일(Firebase Extension) vs 인앱 알림(Firestore 실시간)
